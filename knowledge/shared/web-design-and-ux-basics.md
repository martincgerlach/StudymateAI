# Shared Knowledge: Web Design and UX Basics

## Purpose

This file collects reusable knowledge from 1st semester that can help across multiple assistants.

## HTML and CSS

- HTML is for structure.
- CSS is for visual styling.
- JavaScript is for behavior.
- Semantic HTML should be used when possible.
- `div` should only be used when no semantic element fits.
- Images need correct paths and useful `alt` text.
- Classes help reuse styling.
- Responsive design helps the same site work across screen sizes.

## Visual Design

Important design principles:

- Visual hierarchy.
- Contrast.
- Spacing.
- Typography.
- Gestalt principles.
- CTA clarity.
- Consistent layout.

These principles help users understand what is important and what they can do next.

## UX Research

Basic UX research methods:

- Observation.
- Interview.
- Desk research.
- Survey.

Research should help the designer understand user needs before making decisions.

## Testing

Useful beginner tests:

- 5 second test for first impression.
- Likert scale for measuring agreement.
- Usability test for observing task completion.

Good analysis separates:

- Observation.
- Interpretation.
- Design consequence.

## Project Planning

SCRUM can help structure student projects through:

- Backlog.
- Tasks.
- Sprint.
- Scrum board.
- User stories.
- Short status meetings.

The method should support the project instead of becoming unnecessary administration.
