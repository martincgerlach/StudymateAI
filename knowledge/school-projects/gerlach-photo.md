## Project Type

Gerlach Photo is a photo portfolio website.

The site presents photography through a hero section, navigation, a gallery, categories, and download buttons.

## Project Goal

The goal is to create a simple website where visitors can view photography and navigate between:

- Forside.
- Galleri.
- Om mig.
- Kontakt.

The main value is visual presentation of images.

## Main Features Found

- Hero section with a large image area.
- Navigation bar with logo and page links.
- Gallery page with many image cards.
- Category buttons for Alle, Landskab, Byliv, Portraetter, and Dyr.
- Download links for photos.
- Navy and skyblue visual style.
- CSS grid for gallery layout.
- Hover effect on gallery items.

## Tech Stack

- HTML.
- CSS.
- Image assets.

No JavaScript file was found in the scanned project files.

## Important Design Decisions

### Strong Visual Theme

The site uses a dark navy background with skyblue text and buttons.

This gives the site a clear visual identity, but it should be checked for readability and contrast.

### Gallery Grid

The gallery uses a grid layout with image cards.

This is a good fit for a photography site because images are the main content.

### Download Buttons

Each image card includes a download link.

This makes the gallery more interactive than only showing images.

## Important Code Observations

- The project uses separate HTML pages and one shared CSS file.
- `gallery.html` contains many repeated gallery item blocks.
- The gallery CSS appears more than once with different grid settings.
- `about.html` and `contact.html` were empty in the scanned files.
- Filter buttons exist in the HTML, but no JavaScript file was found to make filtering work.
- The homepage references a hero image path, but the scanned image list should be checked to confirm the asset exists.

## What Works Well

- The idea is easy to understand.
- The visual identity is clear.
- The gallery has real photo content.
- The site uses navigation and multiple pages.
- The image cards and download buttons create a simple user flow.

## Current Weaknesses

- The about and contact pages need real content.
- The gallery HTML is repetitive and could be cleaned up.
- Filter buttons should either work with JavaScript or be removed.
- Some alt text is generic, such as "Foto 1" or "Foto 16".
- The gallery structure appears duplicated, which can make the page harder to maintain.
- Mobile responsiveness should be tested more carefully because the gallery grid is image-heavy.

## Good Exam or Portfolio Explanation

I can explain that Gerlach Photo was a visual website exercise focused on photography, navigation, gallery layout, and simple CSS styling.

I can also explain that the project is not finished yet. The strongest next improvements would be to finish the empty pages, clean up repeated HTML, improve alt text, and make the filter buttons functional with JavaScript.
