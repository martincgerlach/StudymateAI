## Project Type

Portfolio website for showing Martin's web, UX, frontend, and technical problem-solving work.

The project is used for applications, internships, jobs, and project opportunities.

## Project Goal

The goal is to present Martin as a Multimedia Design student with a technical background who can build clear websites, structure content, and explain design and code decisions.

The portfolio should show both:

- Visual and UX thinking.
- Practical frontend implementation.

## Main Features

- Landing page with hero section and clear positioning.
- Selected project cards.
- Case study pages.
- Danish and English language support.
- Dark and light mode toggle.
- CV and material section.
- Contact section.
- Skip link for keyboard users.
- Responsive layout and project screenshots.

## Featured Projects

The portfolio includes several projects:

- LG Bio Capital Partners: client website focused on calm hierarchy, credibility, and content structure.
- LifeScienceNextGen: website in development with information architecture and signup flow.
- Forni Pizza Foodtrailer: client website focused on menu, contact, and mobile readability.
- Blade Rhythm: JavaScript game with state, timing, keyboard input, and difficulty loop.
- AquaShield: school project about clear tsunami emergency information.
- Todo-liste webapp: JavaScript exercise with DOM updates and localStorage.

## Tech Stack

- HTML.

- Vanilla JavaScript.

## Important UX Decisions

### Clear Positioning

The hero text explains that Martin works with web, UX, frontend, and technical problem-solving.

This makes the portfolio more focused than a generic student portfolio.

### Project Cards

Each project card explains:

- Project type.
- Problem.
- Solution.
- Martin's role.
- What the project proves.
- Technologies.

This makes the portfolio easier to scan for employers.

### Case Studies

The LG Bio Capital Partners and Blade Rhythm cases include problem, target group, role, tools, process, decisions, result, and reflection.

This supports portfolio storytelling and exam-style explanation.

### Language Support

The portfolio uses translation data for Danish and English content.

This makes the project more useful for both Danish school contexts and international/professional viewers.

## Important Code Decisions

- The site uses semantic HTML sections.
- CSS variables define colors, spacing, radius, shadows, and layout width.
- JavaScript is used for language switching and theme switching.
- Project content is structured consistently across cards.
- Accessibility basics include skip link, ARIA labels, focus-visible styling, and descriptive image alt text.

## What Works Well

- The portfolio has a clear professional direction.
- The project cards are structured around problem and solution.
- The case studies are stronger than simple project screenshots.
- The bilingual setup shows more advanced frontend thinking while still being explainable.
- The visual style is consistent and polished.
- The project shows a bridge between Multimedia Design and software/frontend direction.

## Current Weaknesses

- Some case studies are more detailed than others.
- The portfolio depends on strong written content, so weak case text would hurt the impression.
- More documented user testing would make the portfolio stronger.
- More before/after evidence would make design improvements easier to prove.
- The site should be checked regularly on mobile because the navigation and project cards contain a lot of content.

## Good Exam or Portfolio Explanation

I can explain that the portfolio is not only a collection of screenshots. It is structured around project purpose, problem, solution, role, and proof.

The main design decision was to make the site calm and clear because future employers need to understand my skills quickly.

The main code decision was to use HTML, CSS, and vanilla JavaScript because the project should be maintainable, explainable, and realistic for my current level.
