# T03 Summary: UX Research, User Types, Moodboards, Testing, and Design Brief

- Observation.
- Interview.
- Desk research.
- Survey.
- User types.
- Empathy.
- Moodboards.
- Lightning demos.
- Design briefs.
- 5 second testing.
- Likert scales.

## Research Methods

The theme presents four basic research methods:

- Observation.
- Interview.
- Desk research.
- Survey.

The important point is that UX decisions should be based on research instead of only personal taste.

## User Types

A target group is not always specific enough.

User types can be described through:

- Needs.
- Goals.
- Skills.
- Experience.
- Habits.
- Values.
- Situation.

This helps design for actual user behavior instead of a broad demographic group.

## Ideation and Visual Direction

Moodboards and lightning demos are used before designing to explore style, inspiration, patterns, and possible directions.

Moodboards help define look and feel.

Lightning demos help find inspiration from existing apps, websites, features, and interaction patterns.

## Testing

### 5 Second Test

A 5 second test checks the user's first impression. It is useful because users often decide very quickly whether they understand a page.

### Likert Scale

A Likert scale measures degrees of agreement. It can be used to test whether a design communicates the intended values.

## Useful Exam Explanation

I can explain that I used research to understand users before making design decisions. I can separate what I observed, what I interpreted, and what I changed in the design.
